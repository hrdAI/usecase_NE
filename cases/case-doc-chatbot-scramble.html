📌 AI 문장 청킹(Chunking)이란?

AI(LLM)가 문장의 구조를 분석하여 '의미 단위(청크)'로 자동 분할하는 기술이에요. 사람이 직접 주어구, 동사구, 목적어구 등을 판단할 필요 없이, 챗봇이 알아서 문장을 나눈 뒤 순서를 섞어(스크램블) 학습 자료를 만듭니다.

1. 성과요약

AI 챗봇 자동화를 통해 10~20개의 영어 문장을 의미 단위(청크)로 나누고 스크램블하는 데 소요되던 수작업 시간을 획기적으로 단축시켰습니다. 생성된 자료는 직독직해 학습용, 워크시트 제작 등 다양하게 활용 가능해졌습니다.

➡️ 영어 학습 자료(청크/스크램블) 제작 시간 대폭 단축!
자동화 이전

📅 소요 시간:

10~20개 문장 수동 작업

👥 작업 방식:

  • PDF 파일 지문을 워드로 복사
  • 문장 단위로 분리
  • 담당자가 직접 문장 분석 (주어구/동사구/목적어구 판단)
  • 의미 단위(청크)로 수동 분할
  • 분할된 청크를 수동으로 스크램블

⚠️ 문제점:

  • 시간 소모적인 수작업
  • 담당자의 주관적 판단에 의존
  • 일관성 부족
  • 반복적인 단순 작업
자동화 이후

⚡ 소요 시간:

즉시 생성 (수초 내)

🤖 작업 방식:

  • 챗봇에게 영어 문장(들) 입력
  • AI가 문장을 의미 단위(청크)로 자동 분할
  • AI가 분할된 청크를 자동 스크램블
  • 결과물 검토 및 활용

✅ 개선 효과:

  • 제작 시간 대폭 단축
  • 일관된 기준으로 청킹
  • 즉시 활용 가능한 자료
  • 콘텐츠 재활용성 증가

2. 개발배경

[ 업무 상황 ]
영어 학습 자료 제작 시, 10개~20개 가량의 영문장을 의미 단위 청크로 하나하나 나누고 스크램블(순서 섞기)하는 작업이 필요했습니다.

[ 비효율 ]
PDF 지문을 워드로 복사한 뒤, 담당자가 직접 영문장을 확인하고 주어구/동사구/목적어구/전치사구로 판단하여 나누는 수작업에 시간이 많이 소요되었습니다.

[ 확장 문제 ]
담당자의 주관적 판단에 의존하여 일관성이 부족하고, 반복적인 단순 작업으로 인한 시간 낭비가 발생했습니다.

업무 시간 비교

  • 다른 워크시트 제작에도 응용하여 활용 가능 (콘텐츠 재활용성 증가)

📌 VBA (Visual Basic for Applications)란?

Excel, Word 등 MS Office 프로그램 내에서 반복 작업을 자동화하고, 맞춤형 기능을 만들 수 있게 해주는 프로그래밍 언어예요. 버튼 하나로 복잡한 HTML 태그 입력을 대신 처리할 수 있습니다.

1. 성과요약

VBA 자동화를 통해 시험지 1개당 30분 소요되던 HTML 태그 작업을 5분으로 단축시켜, 업무 효율성을 6배 증가시키는 성과를 달성했습니다.

➡️ 업무 효율 6배 증가! (30분 → 5분)
자동화 이전

📅 소요 시간:

시험지 1개(30문항)당 평균 30분 이상

👥 작업 방식:

  • 엑셀에서 문항 데이터 확인
  • 밑줄, 이탤릭, 볼드 등 서식 수동 확인
  • 각 서식에 맞는 HTML 태그 수동 입력
  • 어드민 시스템에 등록
  • 등록 후 태그 오류 확인
  • 오류 발견 시 개별 문항 찾아 수정
  • 재등록 및 재확인 반복

⚠️ 문제점:

  • 반복적이고 단순하지만 연구원이 직접 작업
  • 어드민 등록 후에야 오류 확인 가능
  • 오류 수정 및 재확인 과정 반복
  • 휴먼 에러 가능성
자동화 이후

⚡ 소요 시간:

시험지 1개(30문항)당 평균 5분

🤖 작업 방식:

  • 엑셀에서 문항 데이터 작성
  • VBA 매크로 실행 버튼 클릭
  • 서식 기반 HTML 태그 자동 입력
  • 웹 미리보기로 즉시 확인
  • 오류 발견 시 엑셀에서 바로 수정
  • 재실행 후 즉시 재확인

✅ 개선 효과:

  • 시험지당 25분 절약 (30분 → 5분)
  • 어드민 등록 전 오류 확인 가능
  • 오류 수정 과정 간소화
  • 휴먼 에러 제거

2. 개발배경

[ 업무 상황 ]
NELT & 교재 성취 TEST 등 디지털로 구현되는 문항 개발 시 HTML 태그 작업이 필요했습니다. 한 시험지 기준 (평균 30문항) 평균 30분 이상의 시간이 소요되었으며, 이는 반복적이고 단순하지만 연구원이 직접해야 하는 업무였습니다.

[ 비효율 ]
또한, 어드민 시스템에 등록한 후에야 태그 오류 확인이 가능했습니다. 오류가 발견되면 개별 문항을 일일이 찾아 수정하고, 수정 작업 후 재확인 과정을 반복하는 비효율이 발생했습니다.

[ 확장 문제 ]
단순 반복 작업에 연구원의 시간이 소요되어 고부가가치 업무에 집중할 수 없었고, 오류 수정 과정의 비효율로 인해 전체 업무 시간이 증가했습니다.

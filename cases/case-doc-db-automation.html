📌 Excel VBA란?

Excel 내에서 반복 작업을 자동화하는 프로그래밍 언어예요. PDF에서 텍스트를 복사해 엑셀에 붙여넣는 단순 반복 작업을 자동화하여, 콘텐츠 DB 작업 시간을 획기적으로 단축시킬 수 있습니다.

1. 성과요약

Excel VBA 자동화를 통해 1권당 50분 소요되던 수동 '복사+붙여넣기' 작업을 2분으로 단축시켜, 1권당 48분을 절약하고 생산성을 25배 증가시키는 성과를 달성했습니다.

➡️ 1권당 48분 절약, 생산성 25배 증가!
자동화 이전

📅 소요 시간:

1권당 약 50분 소요

(*리딩튜터 주니어 독해집 지문 40개+문항 기준)

👥 작업 방식:

  • 데이터 DB로 정리할 PDF 실행
  • 엑셀 시트에 태깅 항목 설정
  • PDF에 있는 지문과 각 문장의 지시문, 보기 등의 텍스트를 복사
  • 엑셀 시트의 각 태깅 항목에 맞게 붙여넣기
  • 정답 및 해설 부분의 데이터 DB로 정리

⚠️ 문제점:

  • 지문, 문항만 분리 시에도 약 50분 소요
  • 단순 반복 작업으로 시간 소요가 많음
  • 고부가가치 업무에 투입 시간 부족
  • 수작업으로 인한 휴먼 에러 가능성
자동화 이후

⚡ 소요 시간:

1권당 약 2분 소요

🤖 작업 방식:

  • 데이터 DB로 정리할 PDF 실행
  • Excel VBA 매크로 실행
  • 태깅 항목별 텍스트 자동 분리
  • 엑셀 시트에 자동 정리
  • 최종 검토

✅ 개선 효과:

  • 1권당 48분 절약 (50분 → 2분)
  • 생산성 25배 증가
  • 단순 반복 작업 제거
  • 고부가가치 업무 시간 확보

2. 개발배경

[ 업무 상황 ]
콘텐츠 리소스 DB화를 위해서는 태깅 항목별로 분리가 필요한데 현재 사람이 텍스트를 '복사+붙여넣기' 방식의 수동으로 진행하고 있어 시간 소요가 많아 고부가가치 업무에 투입 시간 부족.

[ 비효율 ]
리딩튜터 주니어 독해집 지문 40개+문항 수동 작업 시 약 50분 소요! (*지문,문항만 분리 시)

[ 확장 문제 ]
단순 반복 작업에 많은 시간이 소요되어 콘텐츠 분석이나 기획 등 고부가가치 업무에 집중할 수 없는 상황이었습니다.
